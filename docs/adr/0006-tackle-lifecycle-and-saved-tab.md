# Tackle lifecycle and Saved tab structure

**Tackle** is a commitment action, not navigation alone: it opens the primary source, marks the card as **tackling**, and removes it from the forward feed for the rest of the calendar day. **Save** is a bookmark: the card stays in the feed and is copied to the Saved section. The Saved tab holds two sections — **Tackling** (pinned top, active work) and **Saved** (bookmarked for later) — using compact list rows, not a second snap-paging feed. **Mark done** lives on tackling rows only; tap a row to reopen the primary source. When integrations exist, terminal source states (e.g. PR merged) auto-remove from tackling with user notification and a "still working" override.

**Considered options:** Tackle without hiding from feed; separate In Progress tab; saved-only snap pager; implicit done at end of day.

**Consequences:** Users always have a discoverable home for active work without scrolling back through the feed. Save and tackle can apply to the same card independently.
